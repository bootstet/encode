# 4. 知数减重云健康管理系统（Vue3 + TS + Vite）

项目代码：vita-health-management
角色：核心开发 & TypeScript 改造负责人
技术栈：Vue3、TypeScript、Vite、Ant Design Vue、ECharts、Qiankun、Mock 体系、Commitlint + Husky

项目介绍

面向医院 / 医生团队的减重健康管理系统，支持患者数据管理、健康指标监控、医生团队协作等功能，整体采用 Vue3 + Vite + TypeScript 的现代前端架构，并以微前端方式接入现有医疗业务平台。

主要工作与亮点

  • Vue3 + TS 架构搭建与规范制定：

    • 参与项目整体架构与技术选型，基于 Vue3 + Vite + TS 搭建基础框架，引入 Ant Design Vue、ECharts 与微前端（vite-plugin-qiankun）。
    • 在 tsconfig.json 中启用 strict、noUnusedLocals、noImplicitReturns 等严格配置，并通过 build / build:with-check 区分是否执行类型检查，在保证类型安全的前提下兼顾开发效率。

  • 核心统计模块 TypeScript 改造：

    • 负责 Statistics 统计模块从 JS 向 TS 的迁移（Statistics-TypeScript-Migration.md），梳理并抽象 UserStatisticsData 等核心数据类型。
    • 为 DOM 与 ECharts 实例增加严格类型定义（如 ref<HTMLDivElement | null>、echarts.ECharts | null），并为 Tooltip / Label formatter 定义专用参数接口，减少因 any 导致的运行时错误。

  • Mock 体系与联调效率提升：

    • 设计 VITE_USE_MOCK_DATA 等环境变量驱动的 Mock 数据开关，在开发与测试阶段可一键切换真实接口与 Mock，降低后端依赖与联调阻塞。
    • 统一封装 Mock 数据结构与返回格式，使前端可在后端接口未就绪时完成大部分页面开发与交互联调。

  • 统一请求封装与登录状态管理：

    • 在 utils/request.ts 中实现 401 场景的自动登录处理与页面回跳机制：统一清理登录信息、记录原始路由，在登录成功后自动跳回原页面，提升登录过期场景下的用户体验。
    • 统一接口错误提示与异常上报入口，减少各模块重复处理逻辑，方便后续对接监控系统。

项目价值

  • 为后续医疗业务模块提供一套可复用的 Vue3 + TS 工程模板 和实践指南，包括严格类型配置、多环境构建、微前端接入、Mock 体系等完整链路。
  • 通过核心统计模块的 TS 改造与统一请求封装，在保障业务快速迭代的同时，显著降低联调成本与上线后因类型/登录状态异常导致的缺陷，为团队后续从 Vue2 + JS 向 Vue3 + TS 迁移提供了可参考范例。

# 5. Llm Chat UI 聊天前端项目

项目代码：Llm-chat-ui
角色：前端开发
技术栈：Vue3、TypeScript、Vite、Ant Design Vue、TailwindCSS、Markdown-it、Highlight.js、KaTeX

项目介绍

基于 Vue3 + TypeScript + Vite 的 AI 聊天前端项目，用于对接大语言模型服务，提供对话、上下文管理和富文本展示体验。通过 Ant Design Vue + TailwindCSS 搭建整体 UI 与布局，支持 Markdown、代码高亮与公式渲染，可作为通用 LLM Chat UI 外壳在不同后端模型之间复用。

主要工作与难点 / 亮点

  • 工程化与多环境配置：

    • 从零搭建 Vue3 + Vite 工程，设计包含 dev / test / sit / production 在内的多环境脚本（dev:ft1、dev:prod、build:dev、build:ft1 等），实现开发、联调、预发、线上环境的一键切换与构建。
    • 抽离环境变量与接口基址配置，为后续接入不同大模型服务（自建 / 第三方 API）预留扩展能力。

  • 聊天界面与交互体验设计：

    • 使用 Ant Design Vue 及 TailwindCSS 封装对话布局、消息气泡、输入区、会话列表等通用组件，保证在 PC 端拥有一致的交互与视觉体验。
    • 结合滚动区域自动定位最新消息、发送状态提示等细节优化，提升长对话场景下的可用性。

  • 富文本消息渲染能力：

    • 集成 Markdown-it、Highlight.js、KaTeX，对模型返回的内容进行 Markdown 渲染、代码块高亮与数学公式展示，适配技术问答、算法题解析等复杂场景。
    • 处理 Markdown/HTML 安全性与样式冲突问题，对代码块、列表、表格等元素统一样式，避免主题切换或深色模式下的可读性问题。

  • 类型安全与可维护性：

    • 使用 TypeScript + vue-tsc 对接口响应、消息模型、会话状态等核心数据结构进行类型约束，在开发阶段提前暴露字段缺失或类型不一致问题。
    • 结合严格的 TS 配置和模块拆分，优化组件间 Props / Emits 的类型定义，使聊天记录、配置项在后期扩展（如新增多会话、消息过滤）时更易维护。

  • 可扩展的接口层与错误处理：

    • 设计统一的请求封装与错误处理机制，对网络异常、超时、后端报错、Token 失效等场景给出统一提示，避免影响整体对话体验。
    • 为后续接入流式返回（Server-Sent Events/WebSocket）预留接口层扩展点，为实现“打字机式”流式输出做好前期设计。

项目价值

  • 沉淀了一套可复用的 AI 聊天前端脚手架，新接入一个 LLM 服务时只需替换接口层与部分配置，即可快速构建完整 Chat UI。
  • 在 Vue3 + TS + Vite 组合下实践了从工程搭建、复杂富文本渲染到类型安全与错误处理的完整闭环，为后续在业务项目中落地 LLM 前端提供了可直接复制的参考实现。

# 6. 极客标题 - AI 图片标题生成桌面应用（Electron）

项目代码：electron-geek-title
角色：全栈前端 / Electron 开发
技术栈：Electron、Vue3、Vite、Sharp、Electron Store、Node.js、OpenAI 兼容接口

项目介绍

基于 Electron + Vue3 + Vite 的跨平台桌面应用，面向跨境电商卖家，支持批量读取本地商品图片，通过多家 AI 服务（豆包、阿里云 DashScope、智谱、OpenAI 兼容接口等）自动生成中英文商品标题，并支持过滤、压缩、重命名、CSV 导出等功能。

主要工作与难点 / 亮点

  • 桌面端架构设计与打包发布：

    • 负责 Electron 主进程与 Vue 渲染进程的整体架构设计，拆分 src/main.js（主进程）、preload.js 与 renderer 前端代码，保证安全隔离与通信清晰。
    • 基于 electron-builder 配置 Windows / macOS 打包策略（appId、productName、NSIS 安装向导、macOS 签名开关等），实现一键打包安装包，并兼顾国内网络环境（使用镜像源加速 Electron 依赖下载）。

  • 多 AI 服务商接入与统一抽象：

    • 在 providers/ 中封装 OpenAI 兼容协议层，对接豆包推理接入点、阿里云 DashScope、智谱 GLM-4V 等不同厂商的视觉模型，通过统一的调用接口（模型名称 / Base URL / API Key）屏蔽差异，简化上层业务逻辑。
    • 针对豆包“推理接入点”的特殊配置（ep-xxx 接入点 ID + API Key），编写单独的配置指引与错误排查逻辑，解决模型 404、接入点配置错误等常见问题。

  • 批量图片处理与性能优化：

    • 使用 Sharp 对本地图片进行可选压缩与格式规范化，在保证清晰度的前提下显著降低上传体积，控制 API 调用时延与成本。
    • 设计并实现 可配置的并发队列：支持设置并发数、错误重试、失败任务不阻塞整体进度，并在 UI 中实时展示处理进度、成功率与详细日志，提升大批量任务时的可控性。

  • 标题生成策略与结果管理：

    • 设计支持变量的提示词模板（如 {category}），让用户可以根据品类自定义中英文标题生成规则，提升生成结果贴合业务的程度。

项目介绍

基于 Vue3 + TypeScript + Vite 搭建的聊天前端项目，用于对接大语言模型服务，提供对话、富文本展示等功能。通过 Ant Design Vue 和 TailwindCSS 搭建整体界面与布局，支持 Markdown、代码高亮及数学公式渲染，适合作为 AI 助手 / Chat UI 的通用前端壳。

主要工作与亮点

  • 从零搭建 Vue3 + Vite 工程，配置多环境启动与打包脚本（dev / test / production / sit 等），实现开发、测试、生产环境一键切换与构建。
  • 使用 Ant Design Vue 进行基础组件搭建（布局、表单、按钮、对话区域等），结合 TailwindCSS 进行快速样式编写和主题定制，封装通用布局与消息气泡组件。
  • 接入 Markdown-it、Highlight.js、KaTeX 等库，实现消息内容的 Markdown 渲染、代码块语法高亮和数学公式展示，提升技术类回答和公式类内容的可读性。
  • 使用 TypeScript + vue-tsc 做类型检查，约束接口返回结构与组件 Props 类型，降低联调阶段因类型不一致导致的运行时错误。
  • 结合 Vite 插件与 npm scripts 优化开发体验（例如本地热更新、构建产物分析等），为后续接入真实大模型服务或切换后端接口预留扩展点。

项目价值

  • 形成一套可复用的 AI 聊天前端模板，后续接入不同的 LLM 服务只需调整接口层逻辑即可复用整体界面与交互。
  • 借助 TS 严格模式与工程化配置，为个人 / 团队在 Vue3 + Vite + TS 方向的项目积累了实践样板，便于在正式业务项目中快速复制落地。
