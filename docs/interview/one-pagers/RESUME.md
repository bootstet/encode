### 4. 知数减重云健康管理系统（Vue3 + TS + Vite）  
**项目代码**：`vita-health-management`  
**角色**：核心开发 & TypeScript 改造负责人  
**技术栈**：Vue3、TypeScript、Vite、Ant Design Vue、ECharts、Qiankun、Mock 体系、Commitlint + Husky

**项目介绍**

面向医院 / 医生团队的减重健康管理系统，支持患者数据管理、健康指标监控、团队协作等功能，采用 Vue3 + Vite + TypeScript 的现代化技术栈。

**主要工作与亮点**

- 参与项目整体架构设计与技术选型，搭建 Vue3 + Vite + TS 基础框架，引入 Ant Design Vue、ECharts 和微前端（`vite-plugin-qiankun`）。
- 规划并实施 **TypeScript 严格模式**：在 `tsconfig.json` 中启用 `strict`、`noUnusedLocals`、`noImplicitReturns` 等配置，并通过 `build` / `build:with-check` 区分是否执行类型检查，平衡开发效率与安全性。
- 设计 `VITE_USE_MOCK_DATA` 环境变量驱动的 **Mock 数据开关**，在开发与测试阶段可一键切换真实接口与 Mock，降低后端依赖。
- 负责 Statistics 统计模块从 JS 向 TS 的迁移（`Statistics-TypeScript-Migration.md`）：
  - 为 DOM 与 ECharts 实例增加严格类型（`ref<HTMLDivElement | null>`、`echarts.ECharts | null`）。
  - 为 Tooltip / Label formatter 定义专用参数接口，避免因 `any` 导致的运行时错误。
  - 抽象 `UserStatisticsData` 等数据类型，配合 `vue-tsc` 在编译阶段发现不一致。
- 在 `utils/request.ts` 中实现 **401 自动登录处理与页面回跳机制**：统一清理登录信息、记录原始路由，在登录成功后自动跳回原页面，提升用户体验。

**项目价值**

- 为后续医疗系统模块提供一套可复用的 Vue3 + TS 工程模板。
- 通过严谨的类型系统与 Mock 机制，降低联调成本并减少上线后缺陷。
